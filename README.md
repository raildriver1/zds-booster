# ZDS-Booster

![nslF7moNeUQeEz9qRz0OMe3sGbrpGGdB7kCRRgbIbxyjKgsO2qVDYO3sPJ9uTugC3CQ5a8bptXXbgTXRYy5xeRLB (1)](https://github.com/user-attachments/assets/b6100d33-35af-4a7b-b786-6d65d76c9da1)

Модифицированный движок для **ZDSimulator** на базе DRON's OpenGL Engine v1.1. Подменяет штатный `DGLEngine.dll` и добавляет внутриигровое меню тюнинга, фрикам, расширенный HUD кабины с поддержкой АЛС-ЕН/КЛУБ-У/САУТ/УСАВПП, постпроцесс (FXAA, Bloom), кастомные текстуры неба и собственную физическую модель для **РА-3** поверх движка ЭД4М.

> Полная версия восстановленного движка (без ZDS-Booster) — https://github.com/maisvendoo/DGLEngine

## 🚀 Возможности

Кратко по слоям:

- **Render** — фрикам, тюнинг основной камеры, освещения, FXAA, Bloom, MSAA, анизотропная фильтрация, modern OpenGL context.
- **World** — настройка дальности видимости (провода, дальние модели, светофоры) и новая логика неба с подменой текстур по времени суток и сезону.
- **Locomotive** — патч позиций КЛУБ, расширенный HUD кабины (только ЧС7 целиком), кастомные 3D-тексты в кабине РА-3, RA3Hover (подсветка интерактивных элементов), Developer menu.
- **РА-3** — собственная физическая модель (дизель + гидропередача + реверсор), кастомные модели локомотива, вагона, кабины и индикаторов АЛС-ЕН.
- **КЛУБ-У** — собственный обработчик клавиш с буфером ввода, поддержка команд `К137`, `К122/К123`, `К799/К800`.
- **Конфиг** — `zdbooster.cfg` (общий) и `data/<тип>/<номер>/raildriver/booster.txt` (на локомотив).
- **Локализация** — русский, украинский, английский.
- **Discord Rich Presence** — статус «играю в ZDSimulator».
- **`prikol.py`** — внешний Python-логгер физики РА-3 для отладки.

## ⚙️ Меню F12 (CheatMenu)

Внутриигровое меню — основной способ управлять всеми тумблерами и слайдерами. Реализовано как несколько перетаскиваемых окон с современным тёмным UI (`Source/CheatMenu.pas`, ~4150 строк).

Окна и их содержимое:

### 🎨 Окно «Render»
- **Free Camera** (`freecam`) + раскрываемая секция:
  - `Base speed` — базовая скорость движения фрикама
  - `Fast speed` — скорость с зажатым `Shift`
  - `Turn speed` — чувствительность поворота
- **Main Camera** (`main_camera`) + раскрываемая секция:
  - `Step Forward` — шаг камеры вперёд/назад в кабине
  - `New View Angle` — патч FOV игры + слайдер
  - `Camera Sensitivity` — чувствительность внутриигровой камеры
- **Lighting** (раскрываемая секция):
  - Интенсивность основного света
  - Интенсивность дополнительного света
  - Яркость и контраст кабины
  - Радиус орбиты солнца, высота солнца
- **Brightness** — глобальная яркость

### 🌍 Окно «World»
- **Max Visible Distance** + слайдер — продлённая дальность отрисовки
- **Show Wires** — отрисовка проводов на больших дистанциях
- **Show Distant Models** — дальние модели зданий/деревьев
- **Show Traffic Lights** — дальние светофоры
- **New Sky** — новая логика неба с подменой текстур
- **FXAA** — постпроцесс-сглаживание
- **Bloom** — постпроцесс-блум

### 🚂 Окно «Locomotive»
- **Club Fixes** (`new_club_positions`) — исправление позиций индикаторов КЛУБ на пультах ЭД4М/ЧС7/ЧС8
- **RA3 Hover** — подсветка интерактивных элементов в кабине РА-3
- **Developer Menu** — расширенный режим разработки (доступ к гизмо позиции/поворота/масштаба для кастомных текстов)

### 🧩 Кастомные 3D-тексты в кабине РА-3
В кабине РА-3 можно разместить до 256 произвольных 3D-текстов (источник из переменных КЛУБ/САУТ/УСАВПП), каждый со своим:

- Position X/Y/Z
- Rotation X/Y/Z
- Scale
- Видимость
- Источник данных (`TKlubSource`: скорость, давление, время, координаты, и т.п.)

Редактирование — через гизмо-контроллер: Translate (стрелки), Rotate (кольца), Scale (кубики). Гизмо рисуется даже при закрытом меню F12, чтобы можно было «как РА-3 контроллер» крутить позицию мышью.

Все настройки меню сохраняются в `zdbooster.cfg` (см. ниже) и применяются live, без рестарта игры.

## 🎥 Свободная камера (Freecam)

Реализована в `Source/Modules/DrawFunc3D.pas` через NOP-патч игровой проверки + запись 0/1 в `FREEMODE_SWITCH_ADDR`.

**Управление:**

| Клавиша | Действие |
|---------|----------|
| `Alt + X` | Включить / выключить фрикам |
| `W` / `S` | Вперёд / назад |
| `A` / `D` | Влево / вправо |
| `Пробел` | Подняться |
| `Shift` | Ускорение (использует `Fast speed` из меню) |

При выключении фрикама камера возвращается в исходную позицию. Состояние сохраняется в `freecam: 0/1` в конфиге; при автозагрузке мода фрикам выставляется в нужное состояние.

## 📊 HUD кабины (HookKLUB)

Расширенный визуальный HUD (только для ЧС7 целиком; на других локомотивах работает только часть). Подключается через экспортируемую функцию `HookKLUB`, которую игра зовёт каждый кадр для каждой панели.

Что отображается:

- **УСАВПП (BGSD)** — время, дата, координаты, давление в магистрали, ускорение
- **САУТ** — цифровые индикаторы скорости и допустимых ограничений
- **Уровень ступени тяги (`stupen`)** — текущая позиция контроллера машиниста
- **АЛС-ЕН** — цветные блоки светофора на КЛУБ и БИЛ ПОМЕ
- **Скоростемер** ЧС7 — отдельный hook через `HookSkorostemerCHS7`

Включение элементов делается флагами в `data/<тип>/<номер>/raildriver/booster.txt`:

```ini
saut: 0     # Отображение данных САУТ
bgsd: 1     # Отображение монитора УСАВПП
stupen: 1   # Отображение уровня ступени тяги
```

## 🚦 АЛС-ЕН и КЛУБ-У

Собственный обработчик клавиш КЛУБ-У реализован в `Source/Modules/DrawFunc3D.pas` (state machine с буфером `InputBuffer`). Также есть отдельный модуль `Source/KlubProcessor.pas`, который держит таблицу адресов памяти КЛУБ и оффсетов кнопок (`KLUB_RMP_ADDR`, `BTN_ENTER_CURR`, и т.д. — теперь все собраны в `Source/Modules/Addresses.pas`).

Поддерживаемые команды (вводятся как `K<число>` → `Enter`):

| Команда | Что делает |
|---------|------------|
| `К137` | Активация управления АЛС-ЕН (переход в state 52). После — выбрать частоту 1-3 → `Enter` |
| `К122` | Установить байт по `0x0538D95F` в 1 |
| `К123` | Сбросить байт по `0x0538D95F` в 0 |
| `К799` | Перейти в state 31 (запись DWORD по введённому числу в `0x00749894`) |
| `К800` | Сбросить DWORD по `0x0538D960` в 0 |
| Любая другая | Ошибка — буфер сбрасывается, лог пишет «✗ Неизвестная команда» |

Все события пишутся в `DGLEngine_Log.txt` с префиксами `[К-ВВОД]`, `[КНОПКА СТР]` и т.п. — удобно для отладки.

> ⚠️ HookKLUB (АЛС-ЕН и связанный HUD) сейчас полноценно работает **только на ЧС7 (айди локомотива 822)**.

## 🚆 РА-3: модели и физика

ZDS-Booster содержит самостоятельную реализацию РА-3 поверх движка ЭД4М (`Source/Modules/RA3.pas` + `Source/Modules/RA3Physics.pas`).

### Модели и текстуры
Хранятся в `ra3/`:
- `ra3/loco/` — секции локомотива (Material #N.NNN)
- `ra3/wagon/` — модели вагона (включая колёсную тележку)
- `ra3/cab/` — кабина (material0..material6)
- `ra3/dynamic/` — анимированные элементы:
  - `kontroller.dmd` / `controller_braking.dmd` — рычаг контроллера
  - `arrow_PM.dmd`, `arrow_TC1.dmd`, `arrow_TC2.dmd` — стрелки манометров
  - `button_RB.dmd` / `button_RBP.dmd` / `button_RBS.dmd` — кнопки бдительности
  - `indicator_LS_*.dmd` — индикаторы АЛС-ЕН (G1-G4, R, Y, RY, W)

В составе предусмотрены три секции (локомотив + вагон + локомотив) с интерактивным управлением, поворотом, открытием дверей и анимацией колёсных пар.

### Физическая модель
`RA3Physics.pas` подменяет штатную физику ЭД4М:

- Дизельный двигатель (`disel.cpp` из репо `ra3-equipment` — портирован в Delphi)
- Гидропередача (ГДТ/ГДМ/ГТ — `hydro-transmission.cpp`) с полиномиальной аппроксимацией кривых
- Реверсор
- 10 Гц тик (запись в `LOCSECTIONS[36]` каждые 100 мс)

Управление:

- `Alt + Z` — главный тумблер `ToggleRA3Booster`: одной кнопкой включает физику, разрешает запись в `loc36` и выставляет дефолты (`TractionScale=1.0`, `SpeedLimit=100`). Повторное нажатие — выключает.
- `SetVirtualController(v)` (через UI/гизмо): диапазон `[-5..+5]`, где `0` — нейтраль, `+1..+5` — позиции тяги, `-1..-5` — тормоз. Если задан, он используется вместо штатного байтового контроллера.
- `InstallSpeedHook` / `RemoveSpeedHook` — перехват интегратора скорости (`Launcher.exe + 0x345F00`), позволяет домножать ускорение на `SpeedAccelScale`.

Диагностика выведена через `GetDieselRPM`, `GetM_out`, `GetRailForce`, `GetFillGT`, `GetFillGM`, `GetFillGB`, `GetLocSection36`.

## 🌅 Небо: новые текстуры день/ночь

Включается тумблером **New Sky** в окне «World». Подменяет небосвод собственными текстурами в зависимости от времени суток и сезона.

Куда положить файлы:

```
ZDSimulator/routes/<название_маршрута>/textures/
├── sky_sunriseDawn.bmp           # Предрассветные сумерки (лето)
├── sky_sunsetTwilight.bmp        # Закатные сумерки (лето)
├── sky_sunriseDawn_snow.bmp      # Зимние варианты
└── sky_sunsetTwilight_snow.bmp
```

И модель буста дальности видимости:

```
ZDSimulator/routes/<название_маршрута>/models/
└── sky.dmd
```

Если хотя бы один из четырёх `sky_*.bmp` отсутствует, новая логика неба не активируется — посмотри `DGLEngine_Log.txt`, там пишется отсутствие файлов.

## 🎙️ Discord Rich Presence

`Source/DiscordRPC.pas` подключается к Discord по Named Pipe `\\.\pipe\discord-ipc-0..9`. Отображается активность с timestamp начала сессии. Application ID — `1408466421846905004`. Иконок в текущей версии нет, текстовая активность.

Логи отладки RPC пишутся в `discord_debug.log` рядом с `Launcher.exe`.

## 🛠️ Установка

1. **Резервная копия**: сохрани оригинальный `DGLEngine.dll` из папки игры:
   ```bat
   copy DGLEngine.dll DGLEngine_backup.dll
   ```
2. **Замена**: положи модифицированный `DGLEngine.dll` поверх оригинального.
3. **Первый запуск**: ZDSimulator запустится, при первом старте автоматически создастся `zdbooster.cfg`.
4. **Кастомизация (опционально)**:
   - Положи модели РА-3 из `ra3/` в соответствующие подпапки `data/`
   - Сделай `data/<тип>/<номер>/raildriver/booster.txt` с нужными флагами
   - Положи 4 файла `sky_*.bmp` в `routes/<маршрут>/textures/` для новой логики неба
5. **Перезапуск ZDSimulator**.

## 📝 Конфиг `zdbooster.cfg`

Файл лежит рядом с `Launcher.exe`, формат `key: value`, по одной паре на строку. Парсится в `CheatMenu.pas → LoadConfig`. Полный список ключей:

| Ключ | Тип | По умолчанию | Описание |
|------|-----|--------------|----------|
| `language` | `0/1/2` | `0` | 0 — рус, 1 — укр, 2 — англ |
| `freecam` | `0/1` | `0` | Состояние фрикама |
| `main_camera` | `0/1` | `0` | Тюнинг основной камеры |
| `max_distance` | `0/1` | `0` | Расширенная дальность |
| `newsky` | `0/1` | `0` | Новая логика неба |
| `new_club_positions` | `0/1` | `0` | Патч позиций КЛУБ |
| `ra3_hover` | `0/1` | `0` | Подсветка интерактива РА-3 |
| `new_view_angle` | `0/1` | `0` | Патч FOV |
| `camera_sensitivity` | `0/1` | `0` | Своя чувствительность |
| `show_wires` | `0/1` | `0` | Дальние провода |
| `show_distant_models` | `0/1` | `0` | Дальние модели |

## 🐍 prikol.py — внешний логгер физики

Скрипт на чистом Python 3.x (только `ctypes` + Windows API), читает физические переменные из `Launcher.exe` и пишет в `prikol.log` рядом со скриптом.

**Использование:**

```cmd
python prikol.py            REM обычное логирование (10 Гц, только при изменениях)
python prikol.py -d         REM диагностический дамп памяти + sweep LOCSECTIONS
```

Что читает:

- `kontroller` (byte) — позиция контроллера
- `LOCSECTIONS[36]` (float) — секция, которую перезаписывает физика РА-3
- `revers` (byte/int) — реверсор
- `speed_kph` (float) — текущая скорость

Все адреса вынесены в `addresses.py` — там же лежат оффсеты `OFF_LOCS_36`, `OFF_REVERS`, `OFF_SPEED_KPH`. Если игра обновится и адреса поплывут, правится только `addresses.py`.

> Запускать от администратора, если игра тоже запущена от админа (иначе `OpenProcess` вернёт ошибку доступа).

## 🐛 Диагностика

### Лог-файлы
- `DGLEngine_Log.txt` — основной лог движка (загрузка моделей и текстур, активация подсистем, ошибки инициализации, тип локомотива, события КЛУБ-У и `[К-ВВОД]`)
- `discord_debug.log` — Discord RPC (если включён)
- `prikol.log` — лог `prikol.py` (если запускали)

### Типичные проблемы

1. **HookKLUB не работает**
   - Проверь лог: `Hook activated for locomotive type 822 (CS7)`
   - Если нет — текущий локомотив не ЧС7, для других пока не поддерживается полный HUD
2. **Фрикам не активируется**
   - Проверь `freecam: 1` в `zdbooster.cfg`
   - В логе должно быть `Freecam initialized for locomotive type ...`
   - При неудачном patch видно `ОШИБКА: Не удалось применить NOP patch`
3. **Не загружаются модели**
   - Проверь структуру: `data/<тип>/<номер>/loc/`
   - В логе строка `Модель <название> загружена, ID: <число>`
4. **Не меняются текстуры неба**
   - Должны быть все четыре файла: `sky_sunriseDawn.bmp`, `sky_sunsetTwilight.bmp`, `sky_sunriseDawn_snow.bmp`, `sky_sunsetTwilight_snow.bmp` в `routes/<маршрут>/textures/`
   - В UI должна быть включена «Новая логика неба»
5. **РА-3 физика не запускается**
   - Нажми `Alt + Z` для `ToggleRA3Booster`
   - Проверь, что играешь именно за РА-3 (тип локомотива должен совпадать)
   - Лог покажет включение `SpeedHook` и активацию записи в `LOCSECTIONS[36]`
6. **Discord RPC не подключается**
   - Discord должен быть запущен **до** ZDSimulator
   - Смотри `discord_debug.log` — там пишутся попытки подключения к pipe `discord-ipc-0..9`

## 🔧 HookKLUB — экспортируемые функции

DLL экспортирует две процедуры, которые игра зовёт через call-патч:

```pascal
procedure HookKLUB(
  x: Single;
  y: Single;
  z: Single;
  AngZ: Single
); stdcall; export;

procedure HookSkorostemerCHS7(
  x: Single;
  y: Single;
  z: Single;
  AngZ: Single
);
```

Параметры — позиция и поворот текущей панели (КЛУБ или скоростемера) в системе кабины. Внутри функции по адресам из `Source/Modules/Addresses.pas` читаются текущие значения переменных и рисуются OpenGL-средствами поверх кабины.

Полный список экспортируемых имён DLL — в `Source/DGLEngine.dpr → exports`.

## 🔍 Сборка из исходников

### Требования
- IDE: **CodeGear Delphi 2007** (более новые версии не тестировались)
- Windows 10/11
- (Для запуска полученной DLL) ZDSimulator, OpenGL 1.x+, права администратора (для патчинга памяти)

### Шаги
1. Установи CodeGear Delphi 2007.
2. Открой `Source/DGLEngine.dproj` в IDE.
3. Project → Options → Directories/Conditionals — убедись, что в `Search path` есть `Modules\` и `Modules\Headers\`.
4. Выбери конфигурацию **Release**.
5. Project → Build. Готовая `DGLEngine.dll` появится в выходной папке проекта.
6. Положи её в каталог ZDSimulator поверх оригинальной (с предварительным бэкапом).

### Методы патчинга (для разработчика)
Используется три приёма:
1. **VirtualProtect** — изменение защиты страниц памяти (для записи)
2. **Перехват вызовов** — модификация адреса в `call <addr>` инструкциях, чтобы прыгать в наш код вместо игрового
3. **Инъекция кода** — добавление новых функций через export DLL и подмена адресов в импорт-таблице

Все используемые адреса собраны в `Source/Modules/Addresses.pas` — при обновлении игры именно этот файл нужно править в первую очередь.

## ⚠️ Совместимость и ограничения

- **Версия игры**: мод привязан к конкретному билду `Launcher.exe` (адреса памяти жёсткие). После обновления ZDSimulator адреса могут поплыть и нужно пересобрать `Source/Modules/Addresses.pas` и `addresses.py`.
- **HookKLUB (HUD кабины + АЛС-ЕН)**: полноценно работает только на **ЧС7 (тип 822)**. На остальных локомотивах активны только общие функции — фрикам, новое небо, FXAA/Bloom, тюнинг камеры.
- **Безопасность**: всегда делай резервные копии оригинальных файлов перед заменой DLL.
- **Антивирус**: модификация памяти чужого процесса (`VirtualProtect` + `WriteProcessMemory`) часто триггерит эвристику — добавляй ZDSimulator в исключения, если AV ругается.
